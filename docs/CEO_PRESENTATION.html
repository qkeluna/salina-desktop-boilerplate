Design Patterns

Pattern Purpose Implementation
Domain-Driven Design Organize code by business capability 5 independent domain packages
MVVM (Model-View-ViewModel) Separation of concerns in UI Zustand stores as ViewModels
Repository Pattern Abstract data access SQLite repositories per domain
Service Layer Business logic isolation Pure functions in services/
Monorepo Code sharing & consistency pnpm workspaces with workspace:* protocol

Data Flow

MVVM Data Flow
  1. View (React Component) ‚Üí Triggers action
  2. ViewModel (Zustand Store) ‚Üí Updates state, calls Service
  3. Service ‚Üí Executes business logic, calls Repository
  4. Repository ‚Üí Performs data operations (SQLite)
  5. Model ‚Üí Data returned to ViewModel
  6. View ‚Üí Re-renders with new state
